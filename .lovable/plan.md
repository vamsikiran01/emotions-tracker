
# Add Google Login and Email Notifications

## Overview
Add Google authentication so users must log in before using the app, and send email notifications for login alerts, journal summaries, and streak reminders.

## Prerequisites
- Enable **Lovable Cloud** on the project (Supabase backend)
- Configure **Google OAuth** provider in the Supabase dashboard

## Implementation Steps

### 1. Enable Lovable Cloud
Set up Cloud backend to get Supabase Auth, Edge Functions, and database capabilities.

### 2. Set Up Google Auth Provider
- Guide you through configuring Google OAuth in the Supabase/Cloud dashboard (Google Cloud Console credentials, redirect URLs, etc.)

### 3. Create Auth Pages and Protection

**New file: `src/pages/Login.tsx`**
- A login page with a "Sign in with Google" button
- Uses `supabase.auth.signInWithOAuth({ provider: 'google' })`

**New file: `src/components/ProtectedRoute.tsx`**
- Wraps all app routes so unauthenticated users are redirected to `/login`
- Listens to `onAuthStateChange` for session state

**Update: `src/App.tsx`**
- Add `/login` route
- Wrap existing routes with `ProtectedRoute`

**New file: `src/integrations/supabase/client.ts`**
- Supabase client initialization (auto-generated by Cloud)

### 4. Migrate Storage to Supabase Database

**Create `journal_entries` table:**
- `id` (uuid, primary key)
- `user_id` (uuid, references auth.users)
- `text` (text)
- `result` (jsonb -- stores the EmotionResult)
- `created_at` (timestamptz)
- RLS policies: users can only read/write their own entries

**Update: `src/lib/storage.ts`**
- Replace localStorage calls with Supabase queries
- Filter entries by authenticated user's ID

### 5. Edge Function: Login Alert Email

**New edge function: `supabase/functions/send-login-alert/index.ts`**
- Triggered from the frontend after successful login
- Sends an email to the user's Google email using Supabase's built-in email or a service like Resend
- Content: "You just logged in to Sentira from [device/browser] at [time]"

### 6. Edge Function: Journal Summary and Streak Reminders

**New edge function: `supabase/functions/send-notifications/index.ts`**
- Scheduled via pg_cron (daily)
- For each user:
  - If they have entries from today, compile a brief summary and email it
  - If they missed journaling today, send a streak reminder email
- Requires an email sending service API key (e.g., Resend)

### 7. Secret Setup
- Add email service API key (e.g., Resend) as a secret for the edge functions

## Technical Details

| Component | Technology |
|---|---|
| Auth | Supabase Auth + Google OAuth |
| Database | Supabase Postgres with RLS |
| Email | Resend (or similar) via Edge Functions |
| Scheduling | pg_cron for daily notifications |
| Storage migration | localStorage to Supabase DB |

## Order of Implementation
1. Enable Lovable Cloud
2. Set up Supabase client and Google Auth
3. Create Login page and ProtectedRoute
4. Create journal_entries table with RLS
5. Migrate storage.ts from localStorage to Supabase
6. Set up email service secret (Resend API key)
7. Create login alert edge function
8. Create scheduled notification edge function
